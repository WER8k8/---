
## 五、冲突规避规则

### 5.1 开发边界

| 规则 | 说明 |
|------|------|
| 不越界开发 | 各自负责模块独立开发，不修改对方代码 |
| 接口先行 | 涉及交叉模块，先定义接口契约 |
| 全局变量 | 禁止私自修改全局配置，需双方确认 |
| 数据库 | 表结构修改需 Trae 审核，灵码提交变更申请 |

### 5.2 命名规范

```
前端组件：PascalCase (如 ProductCard.vue)
后端接口：snake_case (如 get_product_list)
数据库表：snake_case (如 product_documents)
变量命名：camelCase (如 productName)
常量命名：UPPER_SNAKE_CASE (如 MAX_RETRY_COUNT)
```
