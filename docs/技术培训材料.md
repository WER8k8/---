## 1. 项目架构概览

### 1.1 整体架构

```
                    ┌─────────────┐
                    │   CDN       │
                    │ (Cloudflare)│
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   Nginx     │
                    │  (HTTPS)    │
                    └──┬─────┬────┘
                       │     │
              ┌────────┘     └────────┐
              │                        │
     ┌────────▼────────┐    ┌─────────▼────────┐
     │  Nuxt.js 3      │    │   FastAPI        │
     │  (Frontend SSR) │    │   (Backend API)  │
     └────────┬────────┘    └────────┬─────────┘
              │                      │
              │            ┌─────────▼─────────┐
              │            │   PostgreSQL 15   │
              │            │   Redis 7         │
              │            │   MinIO           │
              │            └───────────────────┘
              │
     ┌────────▼────────┐
     │  AI Engine      │
     │  (LangChain)    │
     └─────────────────┘
```

### 1.2 模块划分

| 模块 | 路径 | 职责 |
|------|------|------|
| frontend | `/frontend` | Nuxt.js 3 前端应用 |
| backend | `/backend` | FastAPI 后端API |
| ai-engine | `/ai-engine` | AI内容优化引擎 |
| docker | `/docker` | Docker配置文件 |
| scripts | `/scripts` | 部署和运维脚本 |
| docs | `/docs` | 技术文档 |

### 1.3 数据流

```
用户请求 → Nginx → Nuxt.js SSR → Backend API → Database
                                    ↓
                              AI Engine (可选)
                                    ↓
                              返回渲染结果
```

## 5. 开发规范

### 5.1 代码规范

**Python**:
- 遵循PEP 8
- 使用Type Hints
- 函数必须有docstring
- 最大行长度: 120字符

**Vue/TypeScript**:
- 使用Composition API
- 组件名: PascalCase
- Props必须定义类型
- 使用ESLint + Prettier

### 5.2 Git工作流

```
main (保护分支)
  ↑
  ├── feature/xxx (功能分支)
  ├── bugfix/xxx (修复分支)
  └── hotfix/xxx (紧急修复)
```

**提交规范**:
```
feat: 新增关键词排名追踪功能
fix: 修复LLMs.txt生成时的编码问题
docs: 更新API文档
refactor: 重构SEO审计逻辑
```

### 5.3 API设计规范

**RESTful原则**:
- GET: 查询
- POST: 创建
- PUT: 全量更新
- PATCH: 部分更新
- DELETE: 删除

**响应格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {...}
}
```

**错误码**:
- 200: 成功
- 400: 参数错误
- 401: 未认证
- 403: 无权限
- 404: 不存在
- 500: 服务器错误

### 5.4 数据库规范

**命名**:
- 表名: 小写+下划线 (snake_case)
- 字段名: 小写+下划线
- 索引: idx_字段名

**必备字段**:
```sql
created_at TIMESTAMP DEFAULT NOW()
updated_at TIMESTAMP DEFAULT NOW() ON UPDATE NOW()
```

## 6. 故障排查

### 6.1 常见问题

#### 问题1: 服务无法启动

```bash
# 检查日志
docker compose logs -f

# 常见原因:
# 1. 端口被占用
# 2. 环境变量配置错误
# 3. 数据库连接失败
```

#### 问题2: 数据库连接超时

```bash
# 检查数据库状态
docker compose exec postgres pg_isready

# 检查连接数
docker compose exec postgres psql -U youding_prod -c "SELECT count(*) FROM pg_stat_activity;"
```

#### 问题3: Redis缓存失效

```bash
# 清空缓存
docker compose exec redis redis-cli FLUSHALL

# 检查内存
docker compose exec redis redis-cli INFO memory
```

#### 问题4: AI API调用失败

```python
# 检查API密钥配置
import os
print(os.getenv("AI_NVIDIA_API_KEY"))

# 测试API连通性
curl -X POST https://integrate.api.nvidia.com/v1/chat/completions \
  -H "Authorization: Bearer $AI_NVIDIA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"meta/llama-3.1-70b-instruct","messages":[{"role":"user","content":"Hello"}]}'
```

### 6.2 性能问题排查

```bash
# 查看慢查询
docker compose exec postgres psql -U youding_prod -c "
  SELECT query, mean_exec_time, calls
  FROM pg_stat_statements
  ORDER BY mean_exec_time DESC
  LIMIT 10;
"

# 查看API响应时间
curl -w "@curl-format.txt" -o /dev/null -s https://youding.com/api/v1/health
```

### 6.3 安全事件响应

1. **发现异常**: 查看审计日志 `operation_logs` 表
2. **隔离影响**: 禁用可疑账户
3. **溯源分析**: 分析IP、User-Agent、操作记录
4. **修复漏洞**: 根据OWASP指南修复
5. **恢复服务**: 重置凭证、恢复数据
