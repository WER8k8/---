# 系统架构设计

## 一、整体架构方案

### 1.1 前端架构 (Nuxt.js 3)

```
frontend/
├── app/
│   ├── pages/          # 页面路由 (SSR)
│   │   ├── index.vue         # 首页
│   │   ├── products/         # 产品中心
│   │   ├── about.vue         # 关于我们
│   │   ├── news/             # 新闻资讯
│   │   └── contact.vue       # 联系我们
│   ├── layouts/       # 布局模板
│   ├── components/    # 可复用组件
│   │   ├── common/           # 通用组件
│   │   └── seo/              # SEO 功能组件
│   ├── composables/   # 组合式函数
│   ├── stores/        # Pinia 状态管理
│   ├── server/        # API 代理层
│   └── middleware/    # 路由中间件
├── admin/             # vue-vben-admin 后台
│   ├── src/
│   │   ├── views/
│   │   │   └── seo/          # SEO管理模块
│   │   ├── api/              # 后台API
│   │   └── router/           # 后台路由
└── ...
```

## 二、核心设计原则

1. **SSR优先**：所有面向爬虫的页面使用 Nuxt.js SSR 渲染
2. **API统一**：前端通过 Nuxt Server Proxy 调用后端 API
3. **异步处理**：AI 优化、审计等耗时操作使用 Celery 异步执行
4. **缓存分层**：Redis 缓存热点数据，减少数据库压力
5. **安全隔离**：后台管理 (vue-vben-admin) 与前台 (Nuxt.js) 严格权限分离

## 三、数据流设计

```
用户请求 → CDN → Nuxt.js SSR → FastAPI → PostgreSQL
                                        → Redis (缓存)
                                        → AI Engine (异步)
                                        → Crawler (定时任务)
```
