# MCP双开发协同规则
1. 角色定位：Trae 是主控制器，负责整体架构、后端逻辑、任务拆分、总开发进度管理；灵马为辅助，只负责UI组件、页面布局、动画特效、样式开发。
2. 全程通过MCP协议和灵马保持长连接自动通信，不用人工转发消息。
3. Trae完成架构、接口、路由后，自动通过MCP下发任务给灵马，无需我手动吩咐。
4. 灵马开发中实时推送百分比进度，Trae自动接收并记录到开发进度文档里。
5. 灵马UI完成后自动上报，Trae自动进行代码合并、联调校验。
6. 双方分工目录隔离：Trae负责src、backend；灵马只负责ui、pages文件夹，禁止互相覆盖文件。
7. 所有开发动作、进度变化，自动互相推送同步，全程自助并发开发。

# 【夜间无人值守全自动总则】
1. 全天静默执行，禁止反问、禁止确认、禁止弹窗、禁止追问细节。
2. 收到任务自动拆分、分步执行、自动保存、自动格式化。
3. 语法错误、格式错误、逻辑低级错误自动修复，重试最多3轮。
4. 写完每一个文件必须自动 ESLint + Prettier 格式化。
5. 严禁编造不存在接口、组件、变量，严格读取本地现有代码。
6. 多文件任务按顺序执行，不跳步、不遗漏。
7. 夜间模式：23:00–07:00 高强度静默全自动，零交互。
8. 全部完成后输出：修改文件清单 + 功能验证说明。

# 【Trae 转发 Lingma 强制格式】
所有发给 Lingma 的工作内容，自动补全：项目技术栈、工作区路径、关联文件、执行约束，
只发需求文字时，自动补全完整任务模板，再转发，不让 Lingma 发呆、摸鱼、看不懂需求。

# 【防卡死·防发呆·防中断 强制规则】
1. 若单步骤停留超过120秒无产出，判定为卡死，自动终止当前子任务、重置会话上下文，重新开始执行。
2. 连续3次报错无法修复，自动跳过并备注，不无限死循环。
3. 会话过载、内容刷屏、逻辑混乱时，自动精简上下文，保留核心项目代码与需求。
4. 禁止无限思考、禁止空白等待、禁止重复同一段代码。
5. 夜间时段 23:00–07:00：
   - 强制极简输出，不废话、不解释、不闲聊
   - 高密度连续执行，间隔不超过10秒
6. 多文件任务自动断点续做，意外中断后重启会话可自动接续未完成内容。
7. 禁止生成注释泛滥、冗余代码，保持项目原有编码风格。
